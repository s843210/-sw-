## 1. top
시스템의 프로세스 상태를 **실시간**으로 모니터링하는 명령어입니다. 윈도우의 '작업 관리자'와 유사한 역할을 하며, 시스템의 전반적인 리소스 사용량을 한눈에 파악할 수 있습니다.

###  주요 기능
* **시스템 요약 정보:** 시스템 가동 시간(uptime), 사용자 수, 로드 에버리지(load average) 확인
* **리소스 모니터링:** 전체 CPU 및 메모리(RAM, Swap) 사용량 표시
* **정렬 및 관리:** CPU 또는 메모리 사용량 순으로 프로세스 정렬 가능, 실행 중인 프로세스 즉시 종료(k) 가능

###  사용 예시
```bash
$ top
# 종료하려면 'q'를 입력하세요.
```

## 2. ps (Process Status)
현재 시점의 실행 중인 프로세스 목록을  출력하는 명령어입니다. 실시간으로 변하는 `top`과 달리, 명령어를 입력한 순간의 상태를 보여줍니다.

###  주요 기능
* **프로세스 조회:** 현재 실행 중인 프로세스의 PID(프로세스 ID), TTY, 시간, 명령어 확인
* **상세 정보 확인:** 옵션을 통해 사용자(USER), CPU/MEM 사용률, 시작 시간 등 자세한 정보 출력
* **전체 조회:** 시스템의 모든 프로세스를 확인하기 위해 자주 사용됨

###  사용 예시
```bash
# 현재 쉘에서 실행된 프로세스만 간단히 조회
$ ps

# 시스템의 모든 프로세스를 상세한 정보와 함께 조회 (가장 많이 사용)
$ ps aux
# 또는
$ ps -ef
```

## 3. jobs
현재 쉘 세션에서 실행 중인 **작업(Job)의 목록**을 확인하는 명령어입니다. 주로 백그라운드(Background)에서 실행 중이거나 중지된 작업을 관리할 때 사용합니다.

###  주요 기능
* **작업 상태 확인:** Running(실행 중), Stopped(정지됨), Done(완료됨) 등의 상태 표시
* **작업 번호(Job ID) 확인:** `kill` 명령어나 `fg`(포그라운드 전환), `bg`(백그라운드 전환) 명령어에 사용할 작업 번호 확인
* **백그라운드 작업 관리:** `&`를 붙여 실행한 명령들의 상태 추적

###  사용 예시
```bash
$ jobs
[1]+  Running                 sleep 100 &
[2]-  Stopped                 vim test.txt
```

## 4. kill
특정 프로세스에 **시그널(Signal)**을 보내는 명령어입니다. 주로 프로세스를 **종료**하거나 강제로 멈출 때 사용합니다.

###  주요 기능
* **프로세스 종료:** PID(프로세스 ID)를 사용하여 특정 프로세스 종료
* **강제 종료:** 응답하지 않는 프로세스를 강제로 종료 (`-9` 옵션 사용)
* **시그널 전송:** 종료 외에도 프로세스 일시 정지, 재시작 등 다양한 시그널 전송 가능

###  사용 예시
```bash
# PID가 1234인 프로세스에게 종료 시그널(SIGTERM, 15) 전송 (정상 종료 시도)
$ kill 1234

# PID가 1234인 프로세스 강제 종료 (SIGKILL, 9)
$ kill -9 1234

# 작업 번호(Job ID)를 사용하여 종료 (앞에 %를 붙임)
$ kill %1
```
